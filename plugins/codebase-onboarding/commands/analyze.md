# Codebase Analysis

현재 프로젝트의 전체 구조를 분석합니다.

## 분석 항목

1. **프로젝트 타입 식별**
   - 설정 파일로 프로젝트 타입 판별 (Node.js, Python, Go 등)
   - 사용 중인 프레임워크 확인

2. **디렉토리 구조 파악**
   - 주요 디렉토리 역할 분석
   - 소스 코드, 테스트, 설정 파일 위치

3. **엔트리 포인트 확인**
   - 메인 파일 위치
   - 앱 시작점

4. **설정 파일 분석**
   - 환경 변수 요구사항
   - 빌드/실행 설정

5. **문서 확인**
   - README 내용
   - 추가 문서

## 워크플로우

다음 순서로 분석을 진행합니다:

1. 프로젝트 루트의 파일 목록 확인
2. package.json, requirements.txt 등 설정 파일 분석
3. 디렉토리 구조 탐색 (최대 깊이 3)
4. 엔트리 포인트 파일 확인
5. README.md 내용 확인
6. 분석 리포트 생성

## 결과물

분석 완료 후 다음 정보를 포함한 리포트를 제공합니다:

- 프로젝트 개요 (타입, 언어, 프레임워크)
- 디렉토리 구조 다이어그램
- 주요 엔트리 포인트
- 실행 방법 (dev, build, test)
- 환경 설정 요구사항
- 다음 분석 단계 추천
