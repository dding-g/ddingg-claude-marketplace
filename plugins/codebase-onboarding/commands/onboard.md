# Full Onboarding Workflow

새로운 코드베이스에 대한 전체 온보딩 분석을 수행합니다.

## 워크플로우 개요

```
[코드베이스 분석] → [아키텍처 분석] → [의존성 분석] → [컨벤션 분석] → [종합 리포트]
```

## Phase 1: 코드베이스 분석

**Agent**: `agents/codebase-analyzer.md`

- 프로젝트 타입 식별
- 디렉토리 구조 파악
- 엔트리 포인트 확인
- 실행 방법 파악

## Phase 2: 아키텍처 분석

**Agent**: `agents/architecture-analyzer.md`

- 아키텍처 패턴 식별
- 레이어 구조 분석
- 핵심 도메인 파악
- 모듈 의존성 분석

## Phase 3: 의존성 분석

**Agent**: `agents/dependency-analyzer.md`

- 외부 라이브러리 목록화
- 주요 의존성 분류
- 버전 현황 파악
- 학습 필요 라이브러리 식별

## Phase 4: 컨벤션 분석

**Agent**: `agents/convention-analyzer.md`

- 코딩 스타일 파악
- 네이밍 규칙 분석
- 파일 구조 패턴 파악
- 코드 작성 가이드 생성

## Phase 5: 종합 리포트

모든 분석 결과를 종합하여 온보딩 가이드를 생성합니다.

```markdown
# [프로젝트명] 온보딩 가이드

## 1. 프로젝트 개요
[코드베이스 분석 결과 요약]

## 2. 아키텍처
[아키텍처 분석 결과 요약]

## 3. 기술 스택
[의존성 분석 결과 요약]

## 4. 코딩 가이드
[컨벤션 분석 결과 요약]

## 5. 시작하기
- 로컬 환경 설정 방법
- 개발 서버 실행 방법
- 테스트 실행 방법

## 6. 권장 학습 순서
1. [첫 번째 학습 항목]
2. [두 번째 학습 항목]
3. ...

## 7. 유용한 리소스
- [관련 문서 링크]
- [학습 자료]
```

## 실행 모드

실행 모드를 선택할 수 있습니다:

1. **전체 실행** (기본)
   - 모든 Phase 순차 실행
   - 종합 리포트 생성

2. **선택 실행**
   - 원하는 Phase만 선택하여 실행
   - 개별 명령어 사용: `/analyze`, `/architecture`, `/dependencies`, `/conventions`

3. **빠른 분석**
   - 핵심 정보만 빠르게 파악
   - 상세 분석은 생략

## 예상 소요 시간

| 프로젝트 규모 | 예상 시간 |
|--------------|----------|
| 소규모 (< 100 파일) | 2-3분 |
| 중규모 (100-500 파일) | 5-10분 |
| 대규모 (> 500 파일) | 10-20분 |

## 주의사항

- 대규모 프로젝트는 분석 시간이 길어질 수 있습니다
- `.gitignore`에 포함된 파일은 분석에서 제외됩니다
- 민감한 정보가 포함된 파일은 주의해서 다룹니다
