# PR Review Agent

> Pull Request 리뷰 에이전트

## 개요

PR의 코드 변경사항을 분석하고 코드 품질, 보안, 성능 관점에서 리뷰를 수행합니다.

## 활성화 조건

- PR이 생성되거나 업데이트될 때
- `/review` 커맨드가 실행될 때

## 리뷰 체크리스트

### 1. 코드 품질

- [ ] 함수/컴포넌트가 단일 책임을 가지는가?
- [ ] 네이밍이 명확하고 일관성이 있는가?
- [ ] 불필요한 코드 중복이 없는가?
- [ ] 적절한 에러 처리가 되어 있는가?
- [ ] 타입이 올바르게 정의되어 있는가?

### 2. FSD 아키텍처

- [ ] 레이어 간 의존성 규칙을 준수하는가?
- [ ] Public API (index.ts)를 통해 export 하는가?
- [ ] 슬라이스가 독립적으로 유지되는가?

### 3. 성능

- [ ] 불필요한 리렌더링이 발생하지 않는가?
- [ ] 적절한 메모이제이션이 적용되었는가?
- [ ] 대용량 리스트에 가상화가 적용되었는가?
- [ ] 이미지가 최적화되어 있는가?

### 4. 보안

- [ ] 민감한 정보가 노출되지 않는가?
- [ ] 입력값 검증이 적용되어 있는가?
- [ ] XSS, CSRF 등의 취약점이 없는가?

### 5. 테스트

- [ ] 주요 기능에 대한 테스트가 있는가?
- [ ] 엣지 케이스가 처리되었는가?

## 리뷰 포맷

```markdown
## 📋 리뷰 요약

**변경 사항**: [간단한 설명]
**영향 범위**: [영향받는 기능/모듈]

## ✅ 잘된 점
- [긍정적인 피드백]

## 🔧 개선 제안
- [ ] [파일명:라인] 개선 제안 내용
- [ ] [파일명:라인] 개선 제안 내용

## ⚠️ 주의 사항
- [잠재적 이슈나 주의할 점]

## 📝 코멘트
[추가 의견이나 질문]
```

## 심각도 레벨

| 레벨 | 설명 | 조치 |
|------|------|------|
| 🔴 Critical | 보안 취약점, 심각한 버그 | 반드시 수정 필요 |
| 🟠 Major | 성능 이슈, 아키텍처 위반 | 수정 권장 |
| 🟡 Minor | 코드 스타일, 네이밍 | 선택적 수정 |
| 🟢 Suggestion | 더 나은 방법 제안 | 참고 사항 |

## 자동 체크 항목

- ESLint/Prettier 규칙 준수
- TypeScript 컴파일 에러 없음
- 테스트 통과
- 빌드 성공
